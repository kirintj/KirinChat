组件替换策略

移除 Element Plus 后，如何组织原生组件？

A

自建完整组件库

在 src/components/ui/ 下建立一套完整的鸿蒙风格组件库（HButton、HInput、HSelect、HDialog、HMessage 等），统一 API 设计，所有页面切换到新组件。

✅ 架构最干净，可维护性最强   ⚠️ 前期工作量最大

B

逐页替换 + 公共函数

不建独立组件库，直接在每个页面内用原生 HTML + Vue 3 实现所需的 UI 元素，仅抽取 ElMessage/MessageBox 等公共函数为独立工具。

✅ 改动最直接、速度最快   ⚠️ 可能产生重复代码

C

混合：核心组件库 + 页面内联

建 5-6 个高频核心组件（Button、Input、Select、Dialog、Message、Tooltip），其余低频组件（Table、Upload、Tabs 等）在页面内联实现。

✅ 平衡架构与效率   ⚠️ 需要判断哪些算"核心"
