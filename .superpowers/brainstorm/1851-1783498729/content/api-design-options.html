两种 HPageLayout API 设计思路

点击选择你倾向的方案

A

模式驱动（Mode-based）

核心思路：通过 mode prop 切换布局模式（sidebar / list / editor / chat / centered），每种模式有固定的布局结构和可用 slot。

使用示例
<HPageLayout mode="sidebar">
  <template #sidebar>...</template>
  <template #default>...</template>
</HPageLayout>

<HPageLayout mode="list" title="Agent 管理" icon="agent">
  <template #header-actions>...</template>
  <template #default>卡片网格...</template>
</HPageLayout>

优势

  • 每种模式的布局结构明确，不会出现无效组合
  • 内置最佳实践（sidebar 宽度、header 高度等）
  • 新页面接入成本极低，只需选模式 + 填 slot
  • 样式一致性最强

劣势

  • 灵活性受限，特殊布局需求需要 mode 扩展
  • 组件内部逻辑较复杂（5种模式的状态管理）
  • 未来新增布局类型需要修改组件
B

特性组合（Composable features）

核心思路：不设 mode，通过独立的 boolean props 自由组合特性（sidebar / header / scrollable / padded），每个特性独立工作。

使用示例
<HPageLayout sidebar :sidebar-width="280">
  <template #sidebar>...</template>
  <template #default>...</template>
</HPageLayout>

<HPageLayout header padded>
  <template #header>自定义头部</template>
  <template #default>内容...</template>
</HPageLayout>

<HPageLayout scrollable :padding="false">
  <template #default>全屏聊天...</template>
</HPageLayout>

优势

  • 极高的灵活性，任意组合特性
  • 组件更小、更易理解
  • 未来扩展不需要修改核心组件
  • 更符合 Vue 的组合式设计哲学

劣势

  • 可能出现无效或不合理的 prop 组合
  • 各页面需要自行确保一致性（通过约定）
  • 接入成本略高，需要了解每个 prop 的语义
