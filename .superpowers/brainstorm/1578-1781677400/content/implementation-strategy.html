实施策略对比

如此大规模的改造，如何分步推进？

A

分层推进（推荐）

Phase 1: 主题系统 — CSS 变量 + 暗亮切换 + HarmonyOS Sans Symbols

Phase 2: 核心组件库 — Button、Input、Select、Dialog、Message、Tooltip、Form 等

Phase 3: 页面迁移 — 逐页替换 Element Plus 组件为原生组件

Phase 4: 清理 — 移除 Element Plus 依赖、清理残留样式

✅ 每个 Phase 独立可验证   ✅ 主题系统先行，后续开发即见最终效果   ⚠️ 总周期较长

B

按页面逐个完成

以页面为单位，每个页面一次性完成：主题 + 组件替换 + 图标替换。从简单页面（login、404）到复杂页面（agent-skill、mcp-server）渐进推进。

✅ 每完成一个页面就是完整可交付   ✅ 可以边做边验证   ⚠️ 可能产生组件重复实现

C

大爆炸一次性替换

一次性完成所有改动：建主题 → 建组件库 → 替换全部页面 → 移除 Element Plus。在一个大分支中完成。

✅ 没有中间不一致状态   ⚠️ 风险最高，调试困难，review 难度大
