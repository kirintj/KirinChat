实施分期策略

基础层优先 → 壳层组件 → UI 组件 → 页面迁移

Phase 1: 基础设施 (Foundation)

目标：建立鸿蒙设计系统的 CSS 基础
1. Token 同步 — 用参考项目的 harmony-tokens.css + mobile-scale.css 替换现有文件
2. 消除业务别名层 — 删除 style.css 中的 --color-* / --radius-* / --spacing-* / --font-size-* 映射
3. 新增 glass-mixins.css — 液态玻璃效果 CSS 变量
4. HMSymbol 字体引入 — @font-face + 字体文件部署到 assets
5. 全局样式更新 — style.css 精简为全局重置 + transition 动画
产出：所有组件可直接引用 --harmony-* 原生 token，无中间层

Phase 2: 壳层组件 (Shell)

目标：实现标准页面骨架
1. HStatusbar — 从 statusbar-tem.html 提取，PNG 图标资源部署
2. HTitlebar — 4 种变体（big/normal/secondary/drawer），渐隐背板 + backdrop-filter
3. HBottomTab — 6 种变体（2/3/4/5 tab + 1bar + multibar），液态玻璃胶囊
4. HAIBottomBar — AI 底部指示栏
5. App.vue 壳层组装 — 响应式条件渲染（移动端 BottomTab / 桌面端侧边栏）
产出：所有页面共享统一的鸿蒙壳层骨架，响应式自动切换

Phase 3: UI 组件库 (Components)

目标：13 个参考组件 + 现有组件升级
新建（按参考模板）：HSearch · HSwitch · HToolbar · HList · HDivider · HCardView
重构升级：HButton · HChipsTab（对齐参考规范的数值基线 + 交互态）
保留升级：HDialog · HDrawer · HDropdown · HSelect · HForm · HInput · HTabs · HTooltip · HUpload · HAvatar · HSkeleton · HMessage · HScrollbar
HIcon 双模式扩展：HMSymbol Unicode + SVG 降级
产出：完整鸿蒙组件库，每个组件有像素级精确的数值基线和五态交互

Phase 4: 页面迁移 (Pages)

目标：20+ 页面逐步迁移到鸿蒙组件库
按优先级排序，每批迁移后验证：

批次 A（核心页面）：conversation/chatPage · workspace · homepage
批次 B（功能页面）：agent · knowledge · mcp-server · model
批次 C（扩展页面）：interview · voice-interview · dashboard · profile · configuration · tool · agent-skill · mars · construct

每个页面迁移步骤：
① 替换业务变量为 --harmony-* 原生 token
② 替换自定义样式为鸿蒙组件
③ 接入壳层组件（titlebar 变体选择）
④ 响应式验证（桌面 + 移动端）
产出：所有页面完全鸿蒙化，视觉风格统一
