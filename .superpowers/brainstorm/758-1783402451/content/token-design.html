Design Token 体系设计

从间接映射迁移到直接引用鸿蒙原生 Token

现状分析

当前 Token 架构 (3 层间接)
Layer 1: harmony-tokens.css (549 行, 已从参考项目复制)
--harmony-brand, --harmony-font-primary, --harmony-comp-*
↓ @import
Layer 2: style.css (业务映射层) (272 行)
--color-primary: var(--harmony-brand)
--radius-md: var(--harmony-corner-radius-level6)
--font-size-base: var(--harmony-font-size-body-m)
↓ 引用
Layer 3: 组件 CSS
color: var(--color-text-primary) → 间接指向 --harmony-font-primary
问题：业务别名层 (--color-*, --radius-*) 增加维护成本，且与参考规范不同名，不利于对照验证

目标架构

重构后 Token 架构 (2 层直接)
Layer 1: 鸿蒙原生 Token (来自参考项目，直接复制)
harmony-tokens.css — 色彩/字体/交互态/组件色
mobile-scale.css — 间距/圆角/控件高度
glass-mixins.css — 液态玻璃效果 mixin [新增]
↓ 直接引用
Layer 2: 组件 CSS
color: var(--harmony-font-primary) — 直接使用鸿蒙 token 名
padding: var(--harmony-padding-level8) — 无中间层
background: var(--harmony-comp-background-secondary) — 组件色直接引用
改进：消除业务别名层，组件直接使用鸿蒙 token 名，与参考模板一一对应

Token 迁移映射表（关键项）

现有业务变量 → 鸿蒙原生 Token 动作
--color-primary--harmony-brand删除别名
--color-text-primary--harmony-font-primary删除别名
--color-border--harmony-comp-divider删除别名
--radius-md--harmony-corner-radius-level6删除别名
--spacing-md--harmony-padding-level8删除别名
--font-size-base--harmony-font-size-body-m删除别名
--color-bg (白色强制)#ffffff 保留业务约束保留

新增: glass-mixins.css

液态玻璃效果 Token
--glass-blur: blur(20px)
--glass-saturate: saturate(1.8)
--glass-bg: rgba(255,255,255,0.15)
--glass-border: 1px solid rgba(255,255,255,0.2)
--glass-mix-blend: plus-lighter
用于 titlebar、bottomtab、dialog 等浮层组件
