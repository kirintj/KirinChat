# 分布式系统知识点

## CAP 定理
- Consistency（一致性）：所有节点看到相同数据
- Availability（可用性）：每个请求都能得到响应
- Partition Tolerance（分区容错）：网络分区时系统继续运行
- 三者最多满足两个，P 必须保证

## 一致性模型
- 强一致性：写入后立即可读
- 最终一致性：经过一段时间后所有节点一致
- 因果一致性：保证因果关系的顺序

## 分布式事务
- 2PC（两阶段提交）：协调者+参与者，阻塞协议
- 3PC（三阶段提交）：引入超时机制
- TCC（Try-Confirm-Cancel）：业务层补偿
- Saga 模式：长事务编排，补偿回滚

## 分布式 ID
- UUID：简单但无序
- Snowflake：时间戳+机器ID+序列号
- 数据库自增：简单但有瓶颈
- Redis INCR：高性能

## 一致性算法
- Paxos：经典但复杂
- Raft：易理解的一致性算法（Leader选举+日志复制）
- ZAB：ZooKeeper 使用的原子广播协议
